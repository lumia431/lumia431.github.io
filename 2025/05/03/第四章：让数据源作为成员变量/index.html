要支持自定义类型中的 field 成员：

  1. 用户在自定义类型（如 Person）中，针对需响应的字段调用 field 构造；
  2. 当这些 field 字段赋值时，需要触发包含它们的对象整体的 valueChanged；
  3. 为此，在 field 构造时，将 this 指针与对应字段的数据源节点一起注册到一个全局 FieldGraph；
  4. 用户如果用 var 创建自定义类型的数据源，且该类型继承自 FieldBase，则在 var 构建时需检测到已有注册的字段，调度它们观察当前数据源；
  5. 如果用户未通过 var 包装对象，而该对象析构，则在 FieldBase 的析构中，应从 FieldGraph 中移除该对象的所有字段注册。

要实现上述逻辑，需要两大组件：标签基类 FieldBase 和全局单例管理器 FieldGraph。

此外，为了在 var 接收到自定义类型时，识别其继承 FieldBase 并自动调用 setField，建议在 VarImpl 构造中使用 std::is_base_of_v<FieldBase, T> 判断并分发。

decltype(auto) 是 C++14 引入的一个类型推导工具，用于根据表达式的值类型（value category）精确推导变量或返回值的类型。

一、基本作用

它的作用可以理解为：

  • decltype(auto) 会完全照搬表达式的类型，包括是否为引用、是否为 const。
  • 和 auto 不同，decltype(auto) 不会移除引用或 const 修饰。
